Device Import Instructions

1. File Requirements
  1.1 Max 500 rows per import
  1.2 Max 2MB file size
  1.3 CSV or XLSX format

2. Site Fields (* required)
  2.1 SiteModule* – protocol name
  2.2 SiteName – optional
  2.3 Street*, ZIP*, City*, Country* – address fields (required all or none, country's format - ISO 3166-1 alpha-2 (CH, AT, DE...))
  2.4 SIPNumber, SIMNumber, PBXNumber, PSTNNumber - phone number in priority order from most to least prioritized (required at least one, E.164 format, + followed by 10–15 digits)
  2.5 ExternalLink – optional URL

3. Device Fields (* required)
  3.1 EquipmentID* – unique identifier
  3.2 DeviceType* – devic types like: gateway, telealarm, intercom, monitor (see modules list for available types)
  3.3 DeviceModule* – module name (must match DeviceType and SiteModule)
  3.4 DeviceStatus* – device status (Enabled | Disabled)
  3.5 Identity – device identity number
  3.6 Module – device module number for multi-module setups (combination Identity + Module - must be unique per account)
  3.7 Pin – device pin
  3.8 MACAddress – for gateways only (format a1b2c3d4e5f6)
  3.9 IMEI – for gateways only, must be unique (format 359632195095317)
  3.10 FirmwareVersion – optional

4. Sites identification
  4.1 Within the file:
      4.1.1 All phone numbers in a row create the site fingerprint
      4.1.2 Rows belong to the same site only when the entire number set matches exactly
      4.1.3 Any partial overlap (only some numbers match) is a validation error
      4.1.4 SiteName should be left empty on helper rows but the address must still be repeated in every row
  4.2 Matching against existing sites:
      4.2.1 Row attaches to an existing site only when all phone numbers match that site’s numbers
      4.2.2 If numbers partially overlap with an existing site, import is blocked by error
      4.2.3 If numbers match but the address differs, import is blocked by error

5. Module Compatibility
  5.1 SiteModule (protocol) determines which device modules are allowed for corresponding device type (gateway / telealarm / intercom / monitor / ...)
  5.2 Module names must match entries in the modules list below

6. Custom Fields
  6.1 Name custom columns like “FieldName (sitecustomfield)” or “FieldName (devicecustomfield)”
  6.2 Use the same custom field names defined in account settings so values map correctly
